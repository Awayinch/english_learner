# 大作业报告框架

## 题目

《面向个人语言能力构建的垂直知识管理系统开发与实践》

## 1. 研究背景与问题提出

外语学习过程具有明显的知识管理特征。学习者每天接触大量词汇、句型和语境，但这些知识点往往以碎片化方式存在，难以沉淀为可复用、可检索、可复习的个人知识资产。

本项目以 LingoLeap 为基础，将其改造为面向雅思备考的个人知识管理系统，重点解决语言知识的采集、加工、存储、关联和复用问题。

## 2. 需求分析

- 学习者需要快速记录生词和语境。
- 学习者需要看到自己的知识存量和待强化节点。
- 学习者需要将学习结果导出到长期知识库。
- 系统需要兼容 Obsidian 等 Markdown PKM 工具。

## 3. 理论基础

### 3.1 DIKW 模型

系统将离散单词视为 Data，通过词性、释义和例句加工为 Information，再通过标签、双链和复习流程组织为 Knowledge，最后帮助学习者形成真实语言应用能力。

### 3.2 SECI 知识创造螺旋

阅读与对话产生隐性语言经验，生词记录将其外化，Markdown 导出和标签分类实现组合化，复习和测验推动知识内化。

### 3.3 学习分析与词汇习得依据

本项目借鉴学习分析的操作化思路，将可观测学习日志转化为代理指标。词汇学习部分参考词汇知识的 form、meaning、use 框架，将词汇卡片完整率定义为词形、释义、词性和语境证据的组合完整度；同时以语境暴露和主动使用分别对应输入与输出痕迹。

## 4. 研究方法

本项目采用设计型研究与轻量学习分析相结合的方法。系统先在真实学习场景中完成低成本功能改造，再通过本地日志提取可观测指标，用于解释个人语言知识资产的形成过程。

研究数据包括：

- 本地词汇卡片数据。
- 历史对话消息数据。
- 目标词在对话中的出现记录。
- 目标词在学习者输出中的出现记录。
- 学习反思/自评记录，包括认知负荷、学习焦虑、复习信心和下一步计划。
- Markdown 导出的知识卡片。

研究边界包括：

- 当前版本不直接判断学习者是否真正掌握某词。
- 主动回忆和间隔复习需要后续采集测验日志与复习日期。
- 认知负荷、焦虑和信心来自学习者自评，属于过程性主观指标，不等同于最终学习成效。
- 看板数字属于学习过程代理指标，不等同于最终学习成效。

## 5. 系统设计

### 5.1 总体架构

- 交互层：React + Vite 前端界面。
- 学习层：AI 对话、词汇记录、测验。
- 知识加工层：词汇卡片、学习记录、待强化节点。
- 知识存储层：localStorage、GitHub Sync、Markdown 导出。
- 外部知识库层：Obsidian 或其他 Markdown PKM 软件。

### 5.2 新增模块

- 知识资产看板：动态显示词汇卡片、语境暴露、主动使用、卡片完整率和待强化节点。
- 语境自动化增强：为词汇卡片补充同义表达、词根/构词提示、词族候选、自动例句和来源语境。
- 学习反思/自评记录：记录认知负荷、学习焦虑、复习信心和下一步计划，补足自我调节学习证据。
- PKM Markdown 导出：生成中文 Markdown 文件，包含 YAML Frontmatter、双链、标签和增强后的知识卡片。

### 5.3 指标操作化

为避免将装饰性统计误写成学习成效，本项目将看板指标分为两类：

- 已观测指标：由系统已有日志直接计算，包括词汇卡片数、历史消息数、目标词在对话中的出现次数、目标词在学习者输出中的出现次数、卡片字段完整率。
- 待采集指标：当前数据无法支持直接计算，包括主动回忆成功率、间隔复习逾期数、认知负荷与语言学习焦虑等。

这一设计符合学习分析的基本要求，即先定义可观测数据，再解释其作为学习过程代理指标的边界。

## 6. 系统实现

截图建议：

1. 首页聊天界面。
2. 知识资产看板中的“已观测指标”与“待采集指标”。
3. 语境自动化增强后的词汇知识卡片。
4. 学习反思/自评记录模块，展示认知负荷、学习焦虑、复习信心和下一步计划。
5. 点击 PKM 导出的中文 Markdown 文件。
6. Markdown 文件导入 Obsidian 后的双链结构。

## 7. 成果展示

系统实现了从语言学习数据到个人知识资产的转换：

- 生词被结构化为卡片。
- 学习会话被保留为上下文。
- 看板基于真实日志提供知识资产盘点，并显式标注缺测指标。
- 语境自动化增强模块将单一词条组合为更完整的知识卡片，体现知识的组合化与深加工。
- 学习反思模块将主观状态外化为可保存记录，体现自我调节学习中的计划、监控和反思。
- Markdown 导出实现外部化和长期保存。

## 8. 总结与展望

本项目证明了语言学习系统可以被抽象为个人知识管理系统。后续可继续加入间隔重复算法、错题知识图谱、词根词缀自动扩展和 Obsidian 自动同步能力。
