# 大作业报告框架

## 题目

《面向个人语言能力构建的垂直知识管理系统开发与实践》

## 1. 研究背景与问题提出

外语学习过程具有明显的知识管理特征。学习者每天接触大量词汇、句型和语境，但这些知识点往往以碎片化方式存在，难以沉淀为可复用、可检索、可复习的个人知识资产。

本项目以 LingoLeap 为基础，将其改造为面向雅思备考的个人知识管理系统，重点解决语言知识的采集、加工、存储、关联和复用问题。

## 2. 需求分析

- 学习者需要快速记录生词和语境。
- 学习者需要看到自己的知识存量和待强化节点。
- 学习者需要将学习结果导出到长期知识库。
- 系统需要兼容 Obsidian 等 Markdown PKM 工具。

## 3. 理论基础

### 3.1 DIKW 模型

系统将离散单词视为 Data，通过词性、释义和例句加工为 Information，再通过标签、双链和复习流程组织为 Knowledge，最后帮助学习者形成真实语言应用能力。

### 3.2 SECI 知识创造螺旋

阅读与对话产生隐性语言经验，生词记录将其外化，Markdown 导出和标签分类实现组合化，复习和测验推动知识内化。

## 4. 系统设计

### 4.1 总体架构

- 交互层：React + Vite 前端界面。
- 学习层：AI 对话、词汇记录、测验。
- 知识加工层：词汇卡片、学习记录、待强化节点。
- 知识存储层：localStorage、GitHub Sync、Markdown 导出。
- 外部知识库层：Obsidian 或其他 Markdown PKM 软件。

### 4.2 新增模块

- 知识资产看板：动态显示知识存量、学习记录、知识资产、待强化节点。
- PKM Markdown 导出：生成带 YAML Frontmatter、双链和标签的 Obsidian 兼容文件。

## 5. 系统实现

截图建议：

1. 首页聊天界面。
2. 生词本与知识资产看板。
3. 点击 PKM 导出的 Markdown 文件。
4. Markdown 文件导入 Obsidian 后的双链结构。

## 6. 成果展示

系统实现了从语言学习数据到个人知识资产的转换：

- 生词被结构化为卡片。
- 学习会话被保留为上下文。
- 看板提供知识资产盘点。
- Markdown 导出实现外部化和长期保存。

## 7. 总结与展望

本项目证明了语言学习系统可以被抽象为个人知识管理系统。后续可继续加入间隔重复算法、错题知识图谱、词根词缀自动扩展和 Obsidian 自动同步能力。
