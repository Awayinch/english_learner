# PRD: 面向雅思备考的个人语言知识管理系统

## 1. 项目定位

将 LingoLeap 从普通英语学习工具包装为一个垂直领域个人知识管理系统：面向雅思备考，围绕词汇、语境、对话记录、错题与复习节点进行采集、加工、存储和再利用。

## 2. 用户与场景

- 目标用户：需要短期备考雅思的个人学习者。
- 核心场景：阅读或对话中遇到生词，将其沉淀为知识卡片，并导出到 Obsidian 等 Markdown 知识库中复盘。
- 交付目标：在知识管理课程大作业中展示一个可运行系统、可解释功能和可截图成果。

## 3. 需求痛点

传统语言学习常见问题：

- 生词、错题、语境分散在聊天记录、截图、笔记中。
- 词汇只停留在短期记忆，缺少结构化沉淀。
- 难以量化当前知识资产规模和待强化节点。
- 学习记录无法自然进入个人知识库。

## 4. MVP 功能

### 4.1 知识采集

- 聊天与阅读过程中收集生词。
- 支持手动添加词汇。
- 支持 AI 从文本中批量提取词汇。

### 4.2 知识加工

- 为每个词汇保存词性、中文定义、语境例句字段。
- 将词汇视作可复用知识节点。

### 4.3 知识资产看板

- 显示词汇卡片数量：来自本地词汇表存储记录。
- 显示语境暴露事件：统计目标词在历史对话文本中出现的次数。
- 显示主动使用事件：统计目标词在学习者消息中出现的次数。
- 显示卡片完整率：词汇卡片是否具备词形、释义、词性与语境证据。
- 显示待强化节点：缺少释义、缺少语境或仍在待查询队列中的项目。
- 对缺少日志支撑的指标显示“待采集”，不从现有文本中强行推断掌握度。

### 4.3.1 指标边界

本系统当前可以计算的是学习行为日志中的可观测代理指标，例如卡片完整度、语境暴露次数和主动使用痕迹。主动回忆成功率、间隔复习逾期数、认知负荷或学习焦虑等指标需要额外采集测验记录、复习计划和用户自评，不能由当前数据直接推断。

### 4.4 Markdown 知识库导出

- 一键导出 Markdown 文件。
- Markdown 包含中文 YAML Frontmatter 与中文正文结构。
- Markdown 包含 Obsidian 双链，如 `[[词汇/雅思/word]]`。
- Markdown 包含标签，如 `#词汇/雅思`、`#个人知识管理/语言学习`。
- 导出内容可直接作为课程报告截图材料。

### 4.5 语境自动化增强

- 支持一键增强全部词汇卡片。
- 支持对单个词汇卡片进行增强。
- 自动补充同义表达、词根/构词提示、词族候选和 IELTS 场景例句。
- 优先从历史对话中匹配来源语境，无法命中时保留“待补充”状态。
- 增强后的字段会进入页面词卡展示与 Markdown 导出。

## 5. 知识管理理论映射

### DIKW

- Data：原始单词、聊天片段、阅读文本。
- Information：词性、释义、例句、学习场景。
- Knowledge：带标签、双链、语境暴露和主动使用痕迹的 Markdown 知识卡片。
- Wisdom：通过真实日志识别待强化节点，并对缺测指标保持显式标注。

### SECI

- Socialization：阅读与 AI 对话形成语言输入。
- Externalization：将隐性的“不认识”外化为词汇卡片。
- Combination：用同义词、词根、例句、YAML、标签和双链组织为 Obsidian 知识库。
- Internalization：复习、测验和再次使用，使显性知识转化为语言能力。

## 6. 非功能需求

- 低侵入：优先在现有侧边栏增加功能，不破坏聊天主流程。
- 可演示：页面上必须有清晰可截图的看板。
- 可导出：必须能下载 Markdown 文件。
- 可恢复：开发前创建独立 Git 分支，保留回退点。

## 7. 验收标准

- 页面能正常打开。
- 侧边栏出现“IELTS PKM Dashboard”。
- 点击 `PKM` 按钮可下载 `.md` 文件。
- Markdown 文件包含 YAML Frontmatter、DIKW、SECI、词汇双链、标签和指标说明。
- 看板区分“已观测指标”和“待采集指标”。
- 点击“增强”后词卡出现同义表达、构词提示、词族候选和自动例句。
- `npm run build` 通过。
